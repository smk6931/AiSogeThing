[ UI/UX 렌더링 전략: 디바이스별 반응형 처리 ]
작성일: 2026-01-17

1. 개요
   - 데스크탑과 모바일 환경에서 UI/UX가 깨지지 않고 최적화된 화면을 제공하기 위한 기술적 전략 수립.

2. 핵심 전략: CSS Media Query 활용 (Single Codebase)
   - React(JSX) 컴포넌트 구조(HTML 뼈대)는 모바일/데스크탑 간에 동일하게 유지함.
   - 오직 CSS 스타일시트에서 화면 크기(Viewport Width)를 감지하여 레이아웃을 변형시키는 방식을 채택.

3. 상세 구현 방식
   - 기준 해상도(Breakpoint): 768px (일반적인 태블릿/모바일 기준점)
   - 기본 스타일: 데스크탑(Desktop) 기준으로 작성.
   - 모바일 오버라이딩: CSS 파일 하단에 @media (max-width: 768px) 블록을 추가하여, 모바일 환경에서만 적용될 스타일(배치 변경, 크기 조정, 숨김 처리 등)을 정의.

4. 장점
   - JS 로직 분기가 없어 코드가 간결함.
   - 유지보수 시 HTML 구조를 하나만 관리하면 됨.
   - 브라우저 창 크기를 실시간으로 조절해도 즉시 반응함 (빠른 렌더링).

5. 적용 대상
   - 메인 대시보드 레이아웃
   - 네비게이션 바 (가로형 -> 햄버거 메뉴 등)
   - 데이터 테이블 및 카드 리스트 배치 (Grid -> Stack)
