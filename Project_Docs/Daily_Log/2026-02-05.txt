추가 업무 기록: 리액트 vs 게임 엔진 렌더링 최적화
================================================================================

1. 기술 분석: React 상태 관리 vs Game Engine 최적화
   - 주제: 왜 3D 게임 개발에서 useState 대신 useRef를 사용하는가?
   
   - useState (표준 웹 방식)
     * 값이 변경될 때마다 React가 가상 DOM을 비교하여 화면 전체를 재렌더링함.
     * 초당 60회 이상 발생하는 프레임 업데이트(Tick)에 사용 시 과부하 및 렉(Lag) 유발.
     
   - useRef (고속 게임 방식)
     * React의 렌더링 감시망을 우회하여 값을 직접 수정함.
     * React는 값이 변한 줄 모르지만, Three.js(GPU)는 프레임마다 이 값을 직접 읽어감.
     * 결과적으로 CPU 부하를 최소화하면서 매끄러운 60FPS 이동 구현 가능.

2. 구현 사항: 피라미드 펀치 (1단계 분열 로직)
   - 구현 내용: 
     * Unreal 스타일의 Tick 시스템(delta 누적) 적용.
     * 발사 후 1.0초 시점에 현재 위치를 기점으로 +/- 45도 방향 분신 2개 생성.
     * 본체는 방향을 꺾지 않고 원래 속도대로 직진 유지.
     * generation 변수를 통해 자식이 무한 증식하는 것을 방지.

3. 향후 계획
   - 현재 왼쪽 발사체(side:left) 기반으로 로직 완성 후 오른쪽 대칭 적용 예정.
   - 분열 시 펀치들이 겹치지 않게 수직 오프셋(Side Shift) 추가 고려 중.
