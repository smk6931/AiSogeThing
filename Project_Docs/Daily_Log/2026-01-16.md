1. Backend 구조 개편 (Refactoring)
- 기존 routers 폴더 방식에서 도메인 주도 설계(DDD) 스타일로 구조 변경 (hotplace/, youtube/, client/ 등).
- 공통 예외 처리 및 안전한 파일/HTTP 접근을 위한 utils/safe_ops.py 모듈 구현.
- 네이버/유튜브 API 클라이언트를 클래스 방식에서 경량화된 함수형 모듈로 리팩토링.

2. YouTube 기능 구현 (New Feature)
- YouTube Data API v3 연동 완료 (Backend & Frontend).
- 검색(Search) 및 인기 급상승 동영상(Popular) 조회 기능 구현.
- UI: Youtube Lounge 보드 생성 및 Iframe을 이용한 모달 플레이어 구현.
- 403 Forbidden 에러 해결: Python 백엔드 요청 시 Referer 헤더 주입으로 API 보안 우회.

3. API 비용 최적화 (Optimization)
- 비용 절감 전략 수립 및 포트폴리오 문서화 (Project_Docs/Problem_Solving/Youtube_Token_Optimization.md).
- Search API 호출 효율 증대: max_results를 12개에서 50개(최대)로 상향 조정.
- 로컬 파일 기반의 API 할당량(Quota) 카운팅 시스템 구축 및 UI 시각화.

4. Frontend 개선
- API 호출 로직을 컴포넌트에서 분리하여 src/api/ 폴더로 모듈화 (Axios 도입).
- 사이드바 레이아웃 CSS 개선 (.youtube-board 겹침 현상 해결).
