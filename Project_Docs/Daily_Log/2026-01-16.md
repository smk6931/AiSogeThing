[PART 1: 애플리케이션 개발 & 기능 고도화]

1. 콘텐츠 카테고리 세분화 (Dating Extension)
   - 연애 탭 고도화: '연애 예능/코칭'과 '스케치 코미디'로 서브 카테고리 분리.
   - 스케치 코미디 데이터 확보: 숏박스, 너덜트, 180초 등 인기 채널 리스트업 및 RSS 연동.
   - UI 개선: 서브 탭 토글 버튼 및 채널 필터 바(Channel Filter Bar) 구현.

2. AI 기능 지능화
   - 맥락 기반 채널 발굴: 사용자가 보고 있는 탭(예능 vs 스케치)을 인식하여, 해당 장르에 맞는 유튜버만 골라서 AI가 찾아내도록 로직 개선.
   - 검색 API 활용 효율 극대화 (정확도 향상).

3. 백엔드(DB) 기초 공사
   - database.py: SQLAlchemy 엔진 및 세션 설정 (SQLite).
   - models.py: User(회원), Comment(댓글), YoutubeLog(시청기록) 모델 정의.
   - 배포 및 확장 계획 수립: 댓글 시스템 및 취향 매칭(Interest Graph) 데이터 설계를 위한 기획(IdeaNote.md).

4. 시스템 최적화
   - JSON 기반 데이터 관리: 동적으로 늘어나는 채널 리스트를 dating_channels.json 파일에 카테고리 태그(category)와 함께 저장/관리하도록 구조 변경.
