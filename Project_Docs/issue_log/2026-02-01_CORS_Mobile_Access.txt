=== 2026-02-01 모바일/외부 기기 접속 및 CORS 이슈 정리 ===

1. 이슈 현상
   - 내 휴대폰에서는 로그인/회원가입이 되는데, 다른 사람 휴대폰에서는 "회원가입 실패" 발생.
   - DB에는 데이터가 생성되지만 프론트엔드에서 에러로 처리됨.

2. 원인 분석
   - CORS(Cross-Origin Resource Sharing) 보안 정책 위반.
   - 서버는 요청을 처리(200 OK)하지만, 브라우저가 응답 헤더(Access-Control-Allow-Origin)를 확인하고 차단함.
   - 백엔드 설정(`allow_credentials=True`)상 `allow_origins=["*"]` 와일드카드가 무효화됨.
   - "내 폰"은 localhost로 인식되거나 보안 예외로 통과되었으나, "다른 폰"은 외부 Origin으로 인식되어 차단됨.

3. 해결 방법 (환경별)
   A. 로컬 개발 환경 (노트북 + 같은 와이파이 폰)
      - Front: `vite --host`로 실행하여 내부 네트워크(IP) 접속 허용.
      - Front .env: API 주소를 `localhost`가 아닌 내부 IP(`172.30.1.77`)로 지정.
      - Back: CORS 설정에 내부 IP 추가.

   B. 클라우드 서버 환경 (현재 사용자 상황)
      - Front: 별도 IP 설정 불필요 (배포된 도메인/Public IP 사용).
      - Back: 불특정 다수의 접속을 허용해야 하므로 `allow_origins` 리스트 방식보다는 정규식(`allow_origin_regex`) 사용 권장.
      
4. 조치 사항
   - 불필요해진 로컬 IP(172.30.1.77) 관련 설정 제거.
   - 클라우드 환경 대응을 위해 백엔드의 `allow_origin_regex` 설정은 유지 (모든 출처 허용).
