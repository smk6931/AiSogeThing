[ 1. 프로젝트 개요 (Project Overview) ]
--------------------------------------------------------------------------------
* 프로젝트 명: SogeThing (Social + Thing / 소개팅 중의적 의미)
* 핵심 컨셉: 사용자 채팅 스타일과 콘텐츠 소비 성향을 AI로 분석하여, "결이 맞는" 사람을 매칭해주는 소셜 서비스
* 개발 목표:
    - 서비스: 실제 운영 가능한 수준의 고품질 매칭 서비스 구현
    - 포트폴리오: 대기업/중견기업 취업을 위한 "엔터프라이즈급 아키텍처 및 DevOps 역량" 증명
* 타겟 유저: 외모나 단순 스펙보다 대화 통화, 관심사가 비슷한 사람을 찾는 사용자


[ 2. 주요 기능 (Key Features) ]
--------------------------------------------------------------------------------
A. 사용자 성향 분석 (AI Core)
   - 채팅 분석: 사용자의 대화 톤, 매너, 주요 키워드 분석 (LLM 활용)
   - 콘텐츠 소비 분석: 좋아요/공유한 콘텐츠(뉴스, 유튜브, 글 등)를 기반으로 관심사 태깅
   - 페르소나 생성: 위 데이터를 종합하여 사용자의 "Digital Persona" 생성

B. 매칭 시스템 (Matching Engine)
   - Vector Search: 사용자 페르소나 임베딩을 통한 유사도 매칭
   - 궁합 리포트: 
       "이분과는 대화 티키타카가 잘 될 확률 85%!"
       "두 분 다 'SF 영화'와 '재테크'에 관심이 많으시네요."

C. 커뮤니티 & 소셜
   - 1:1 안심 채팅: 초기에는 AI가 대화 주제 추천 (Ice Breaking)
   - 관심사 피드: 매칭된 관심사 관련 콘텐츠 공유


[ 3. 인프라 및 배포 전략 (Infrastructure Strategy) ]
--------------------------------------------------------------------------------
Q. 로컬 개발 vs AWS 선배포?  
A. 결론: Docker 기반 로컬 개발 -> (CI/CD) -> AWS 자동 배포 전략 추천 (Hybrid)

1. 로컬 개발 후 배포 (일반적인 방식)
   - 장점: 비용 0원, 초기 개발 속도 빠름.
   - 단점: 나중에 "로컬에선 되는데 서버엔 안 돼요" 문제 발생 가능.

2. AWS 바로 시작 (클라우드 네이티브)
   - 장점: 처음부터 실서버 환경 고려, 포트폴리오에 강력함.
   - 단점: 비용 발생, 설정 시간 소요.

3. ✅ 추천 전략: Hybrid (Dockerize First)
   - 개발 환경: 로컬에서 개발하되, 모든 컴포넌트(DB, API, Frontend)를 "Docker Compose"로 띄워서 개발.
   - 효과: 로컬 환경이 곧 서버 환경과 동일해짐.
   - AWS 전환: Docker 이미지만 AWS(EC2)에 올리면 끝.
   - 취업 포인트: "Docker 컨테이너 환경 구축 및 CI/CD 경험" 강조 가능.


[ 4. 기술 스택 (Tech Stack - Tentative) ]
--------------------------------------------------------------------------------
* Backend: Python (FastAPI or Django) - AI 라이브러리 연동 용이
* Frontend: Next.js (React) - SEO 및 PWA, 트렌디한 디자인
* AI/ML: LangChain / LangGraph, Vector DB (Pinecone or Chroma)
* Database: PostgreSQL (RDB), Redis (Cache/Chat)
* DevOps: Docker, Github Actions, AWS (EC2, RDS)


[ 5. 단계별 로드맵 (Roadmap) ]
--------------------------------------------------------------------------------
* Phase 1 (기획 & 설계): 요구사항 정의, DB 스키마 설계, API 명세
* Phase 2 (MVP 개발 - 로컬): Docker 환경 세팅, 핵심 기능(분석/매칭) 구현
* Phase 3 (배포 및 고도화): AWS 인프라 구축, CI/CD 연결, 실제 베타 테스트
