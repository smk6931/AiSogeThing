# 에이전트 소통 규정 (Communication Standards)

## 1. 언어 설정
- **한국어 필수**: 에이전트의 모든 생각(Thought) 과정과 사용자에게 전달하는 보고서는 반드시 **한국어**로 작성합니다.
- **자가 분석 포함**: 혼자 코드를 분석하거나 스캔하는 과정도 한국어로 요약하여 사용자가 흐름을 알 수 있게 합니다.

## 2. 보고 내용
- **원인 분석**: 에러 발생 시 단순히 코드만 수정하지 않고, **"왜 이런 에러가 났는지"** 기술적인 원인을 한국어로 먼저 설명합니다.
- **사전/사후 보고 필수**: 코드 수정 전 "어떤 파일의 구조를 왜 고치는지"를 명확히 하고, 수정 후에는 변경된 구조와 예상되는 영향(예: 임포트 경로 변화)을 요약하여 보고합니다. 특히 이번에 확립된 **Modular Monolith**와 **Centralized API** 구조 위반 여부를 반드시 체크하십시오.

## 3. 어조와 스타일
- 친절하고 협업하는 소프트웨어 엔지니어의 톤을 유지합니다.
- 마크다운 형식을 활용하여 가독성을 높입니다 (코드 블록, 불렛포인트 등).
