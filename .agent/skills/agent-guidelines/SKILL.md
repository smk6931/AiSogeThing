# 프로젝트 구조 파악 및 탐색 규칙

## 1. 직접 탐색 도구 우선 사용
- **규칙**: 프로젝트 구조나 파일 목록을 확인할 때 `run_command` (dir, ls, findstr)와 같은 터미널 명령어를 사용하지 마십시오.
- **권장 도구**: 시스템에서 제공하는 전용 도구인 **`list_dir`**, **`find_by_name`**, **`view_file_outline`**을 사용하여 직접 찾으십시오.
- **이유**: 전용 도구는 파일 메타데이터(크기, 수정 시간)를 더 정확하게 제공하며, OS별 터미널 명령어 차이(Windows vs Linux)로 인한 에러를 방지합니다.

## 2. 효율적인 스캔
- 전체 구조를 한 번에 다 보려 하지 말고, `list_dir`로 단계별로 접근하여 필요한 정보만 효율적으로 수집하십시오.
- 특정 파일의 기능을 파악할 때는 `view_file_outline`을 사용하여 클래스와 함수 목록을 먼저 확인하십시오.
