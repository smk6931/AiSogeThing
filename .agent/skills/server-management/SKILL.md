# 서버 실행 및 관리 지침

## 1. 서버 실행 금지
- **원칙**: 에이전트는 프론트엔드(`npm run dev`)나 백엔드(`uvicorn`, `python main.py`) 서버를 직접 실행하여 서비스를 시작하지 않습니다.
- **예외 (직접 실행 허용)**: 파일 이동(`Move-Item`, `mv`), 디렉토리 생성(`New-Item`, `mkdir`), 파일 삭제 등 **프로젝트 구조 관리 및 코드 리팩토링과 관련된 명령어**는 작업 효율을 위해 에이전트가 직접 `run_command`의 `SafeToAutoRun: true` 옵션을 사용하여 실행할 수 있습니다. 이미 실행 중인 서버를 끄는 명령(`Stop-Process`, `Terminate`)도 직접 실행 가능합니다.

## 2. 서버 종료 방법 (에이전트가 띄운 경우)
- 에이전트가 `run_command`로 서버를 실행한 경우, 다음 단계를 통해 종료할 수 있습니다.
    1. `send_command_input` 도구를 사용합니다.
    2. `Terminate: true` 옵션을 설정하여 프로세스를 강제 종료합니다.
    3. 사용자에게 서버가 종료되었음을 보고합니다.

## 3. 사용자 안내
- 사용자가 직접 띄운 서버는 터미널에서 `Ctrl + C`를 누르거나 터미널 창(휴지통 아이콘)을 닫아 종료할 수 있음을 안내합니다.
