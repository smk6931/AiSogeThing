# 게임 개발 코딩 가이드 (Game Development Standards)

## 1. 투사체(Projectile) 구현 원칙
- **단일 책임 원칙:** 투사체 컴포넌트(예: `Projectile.jsx`)는 오직 **"주어진 방향과 속도로 이동"**하는 역할만 수행한다.
- **언리얼 스타일:** `ProjectileMovement` 컴포넌트처럼 초기 위치(`startPos`)와 속도(`velocity`) 혹은 방향/속도 조합을 받아 독립적으로 움직이도록 설계한다.
- **로직 분리:** 분열(Split), 유도(Homing), 특수 효과 등 복잡한 비즈니스 로직은 투사체가 아닌 **스킬 서비스(`usePlayerSkills.js`)**나 **매니저 훅(`useProjectiles.js`)**에서 제어한다.

## 2. 주석 및 파라미터화 규칙
- **쉬운 수정:** 사용자가 코드를 직접 수정하기 쉽도록 핵심 파라미터는 변수로 빼고, 상단에 명확한 주석을 단다.
- **주석 예시:**
  ```javascript
  // [파라미터 조절용]
  const speed = 5.0;       // 투사체 날아가는 속도 (클수록 빠름)
  const count = 3;         // 한 번에 발사되는 개수
  const angleRange = 45;   // 발사 각도 범위 (도 단위)
  ```
- **한글 설명:** 한국어 사용자를 위해 모든 파라미터와 주요 로직 블록에는 한글로 상세 설명을 추가한다.

## 3. 스킬 설계 가이드
- **데이터 중심:** 스킬의 "모양"을 결정하는 로직(예: 원형 발사, 일직선 발사 등)은 수학적 계산을 통해 미리 좌표를 산출한 뒤 투사체를 생성한다.
- **분열 시스템:** 투사체가 스스로 분열하는 대신, 특정 시간이나 조건이 되면 부모 레이어에 '분열 요청' 이벤트를 보내고 부모가 새로운 투사체를 소환하도록 유도한다. (중앙 집중 제어)
